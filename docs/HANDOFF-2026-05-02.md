## Lessons Learned (für künftige Sessions wertvoll)

1. **Backend-Allowlist als Sicherheitsnetz**: Edge Function `update_prize` listete
   bereits ALLE potentiell relevanten Felder als erlaubt — Frontend konnte erweitert
   werden ohne Backend-Deploy. Pattern für sichere Schnittstellen-Evolution.

2. **OAuth-Redirect-Allowlist + Site URL fallback**: Wenn `redirect_to`-URL nicht
   in der Allowlist → Supabase fällt **stillschweigend** auf Site URL zurück.
   Site URL muss aktuell sein; lokales Testen erfordert localhost in Allowlist.

3. **Inactive-by-default-Pattern für Sentry**: `if (!DSN) return` am Anfang von
   `init()`. Tree-Shaking entfernt Sentry-Bundle wenn nicht aktiv. Aktivierung
   ohne Code-Änderung möglich (nur Secret setzen + re-deploy).

4. **Time-based Aktivierung statt Toggle**: Wochenpreise haben keinen
   `is_active`-Flag, sondern `week_start = aktueller_montag` als zeitlichen Filter.
   Reduziert Admin-Aufwand: kein „aktivieren / deaktivieren"-Klick, einfach Datum
   passend setzen.

5. **share-page-Pattern für SPA + LinkedIn**: SPA kann meta-tags nicht crawler-
   sichtbar pro User personalisieren. Edge Function als HTML-Generator löst das,
   Mensch wird per JS-Redirect zur SPA geleitet.

6. **Module-Cache statt Context-Provider**: Bei 1 Setting-Set + niedrigerer
   Frequenz (1× pro Tab) ist `let cache = null` im Modul + `useState` im Hook
   einfacher als Context-Provider. Invalidate-Funktion explicit exportieren.

7. **CSS-Variablen-Override für Quiz-Bereich**: Statt globale Farbe ändern und
   überall durchschlagen lassen, separate Variable `--color-text-quiz` und nur
   in 5 Quiz-Files via Klassen-Rename ersetzen. Klare Scope-Trennung.
