## Architektur-Stand nach heutiger Session

### i18n-System

- **`useLocale.tsx`** (Context-Provider): zentral, Singleton via React Context
- **`lib/constants.ts: lf()`**: `lf({de, en, tr?, es?}, locale)` mit EN-Fallback
- **Lokales Pattern pro Page**: `const L = { key: { de, en, tr, es } }` am
  Datei-Anfang, dann `lf(L.key, locale)` an Verwendungsstellen
- **`lfFn()`** für parametrisierte Strings: `lfFn(L.betterThan, locale, percent, total)`
- **`lib/i18n.ts: getI18nText()`** für DB-JSONB-Felder mit Fallback-Kette
  `locale → en → de → fallback`
- **`useAuth.ts: syncLocaleToProfile()`**: User-Wahl in localStorage gewinnt gegen
  DB-Default 'de' aus dem `handle_new_user`-Trigger

### Edge Functions (relevante Änderungen)

- **`admin`** Edge Function:
  - `list_users` joint jetzt auf `auth.users` für E-Mails (via service-role
    `db.auth.admin.listUsers`)
  - `update_prize` Action neu (für i18n-Updates aus dem Admin-UI)
- **`get-sponsors`**: liefert `description_i18n` + `title_i18n` aus

### Mojibake-Problem

Beim Update von Sonderzeichen via Supabase Dashboard SQL-Editor wird oft doppeltes
UTF-8-Encoding produziert (`für` → `fÃ¼r`). Convert_from-Trick funktioniert nicht
zuverlässig.

**Workaround**: Daten-Updates mit Sonderzeichen IMMER über das Admin-UI im Browser
machen — `fetch` mit `JSON.stringify()` enkodiert garantiert sauber UTF-8, Edge
Function speichert korrekt.

Memory-Eintrag: `feedback_supabase_schema_cache.md` (für Schema-Cache-Outage),
`feedback_supabase_db_push.md` (DB-Migrations via Dashboard SQL-Editor).

## Lessons Learned (für künftige Sessions wertvoll)

1. **Schema-Cache-Outage zuerst diagnostizieren**: Dashboard → Settings → API →
   Exposed Schemas. Wenn `api` an erster Position oder `public` fehlt → das ist
   der Bug. Spart Stunden Code-Debugging.

2. **Daten-Updates mit Sonderzeichen NIE über SQL-Editor**: Browser-fetch +
   Edge Function ist UTF-8-sauber. SQL-Editor produziert Mojibake bei
   Türkisch/Spanisch/Umlauten.

3. **useAuth race condition Pattern**: `mounted = true/false` flag + try/finally
   für setLoading + 8s safety-timer als last-resort fallback. Gilt für alle
   async-init in React-useEffect.

4. **Local State für DB/localStorage-Sync-UIs**: Toggles die externe Quellen
   lesen (DB, localStorage, browser API) brauchen lokalen React-State + sync
   via useEffect. Sonst rerendert die Komponente nicht nach Update.

5. **Tailwind-Variable-Override**: Text-Farbe einmal in `--color-text-primary`
   ändern wirkt auf 70+ Stellen sofort. Nur `text-white` direkt umgeht es —
   muss separat gefixt werden.

6. **Bulk-Refactor via Python-Script**: Bei 30+ identischen `locale === 'de' ?`
   Patterns ist ein temporäres Python-Script massiv schneller als Edit-für-Edit.

7. **L-Pattern für i18n**: `const L = { key: { de, en, tr, es } }` pro Page
   skaliert besser als 30+ inline `lf({de,en,tr,es})`-Objekte und ist trotzdem
   lokal — kein Round-trip zu globaler i18n-Datei.
